What is XSS?

Cross-site scripting (XSS) is a security bug that can affect websites. XSS can allow an attacker to add their own malicious JavaScript code onto the HTML pages displayed to your users. Once executed by the victim's browser, this code could then perform actions such as completely changing the behavior or appearance of the website, stealing private data, or performing actions on behalf of the user.

XSS Prevention

  • Validate all user provided content
  • Encode data before inserting
    • <b>hello</b> == &#x3C;b&#x3E;hello&#x3C;/b&#x3E;
  • Sanitize using proven library
  • Use modern frameworks when possible
  • HttpOnly flag to prevent session hijacking

What is CSRF?

Cross-Site Request Forgery (CSRF) is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated.

CSRF Overview

  • CSRF attacks typically targeted - specific website, endpoint, user
  • Attacker cannot read responses - endpoints must be state-changing
  • Requirements
    1. Web Application must use cookie based authentication
    2. User (victim) must be logged in to target website
    3. User must click on malicious link
